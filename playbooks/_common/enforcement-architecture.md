## Regla: Worktree Session Hygiene

Los PreToolUse hooks `enforce-routing.sh`, `enforce-secrets.sh`, `enforce-self-mod.sh` y `enforce-subagent-isolation.sh` distinguen main session de subagente vía **marker-file registry** (`.claude/.subagent-active.json`) resuelto por `.claude/hooks/lib/role-detection.sh`. El heurístico anterior basado en `cwd` fue eliminado por PR #358 (root-cause fix: main session ejecutando dentro de un worktree era clasificada erróneamente como subagente). Para evitar falsos positivos/negativos:

1. **Arrancar `claude` SIEMPRE desde el worktree donde vas a trabajar**, no desde main repo. `cd .claude/worktrees/<branch> && claude`. Motivo: el scope default del marker al dispatchar subagentes = cwd de la sesión. Si arrancás desde main repo, scope=main repo, atrapando a sesiones en cualquier worktree descendiente (walk-up trap). Desde el worktree, scope=worktree, no contamina paralelos. Bug estructural de role-detection tracked en VI-186.
2. **Subagentes registran marker al iniciar**. La Capa de aislamiento los confina al worktree donde se registraron, independientemente del `cwd` que reciban.
3. **Main session sin marker** queda clasificada como `main` por defecto (fail-safe). Puede operar desde repo root o desde un worktree sin perder el rol.
4. **Nunca** hacer `cd $WORKTREE_PATH` desde la main session para editar archivos protegidos — escalar al subagente apropiado. Lectura de Class A (services/**, packages/**, infra paths) desde main **sí permitida** (routing no enforcea Read); lectura de secretos (.env*, secrets/**, *.pem, *.key, *credential*) está bloqueada para todos los roles por `enforce-secrets`.
5. **Worktrees paralelos/anidados**: el subagente se confina al scope registrado en su marker — no puede escribir ni leer fuera de ese scope (ni ancestros, ni worktrees hermanos, ni scopes de otros subagentes).

Diseño conservador: `AIDLC_ENFORCEMENT_MODE=hard` es default; `disabled` permite bypass total; `warn` **NO se honra** en los hooks de enforcement (sería downgrade silencioso de hard-denies). Solo `disabled` desactiva enforcement.

### Enforcement Layers

The post-implementation chain is backed by **5 structural enforcement layers** (L_path, L2, L4a, L4b, L5) plus **L3 cleanup** (soft, behavioral). The structural layers guarantee correct behavior; L3 only releases the marker after a subagent finishes. Note: prior to #34 this section claimed L3 was structural — corrected; the actual structural chain enforcement happens at commit time (L4b).

| Capa | Mecanismo | Tipo | Propósito |
|---|---|---|---|
| **L_path** (4 hooks SRP) | PreToolUse: `enforce-routing.sh`, `enforce-secrets.sh`, `enforce-self-mod.sh`, `enforce-subagent-isolation.sh` (VI-137 consolidó L1+L6+L7+L8; PR #358 decompuso el god-object en SRP) | Hard | Marker-based role detection (`lib/role-detection.sh` — reemplaza el heurístico cwd-broken). Role = `subagent` si marker fresco en `.claude/.subagent-active.json` walk-up desde cwd; si no, `main` (fail-safe). **enforce-routing** (Edit/Write/Bash; Read NO): main no **edita/escribe** Class A (`services/**`, `packages/**`, `.github/workflows/**`, `Dockerfile*`, `docker-compose*`, `Makefile`, `scripts/**`) — fuerza dispatch a typed agent. Extras bash: `git apply` desde main siempre bloqueado (INFRA-H2); interpretadores-as-editor (`python -c`, `node -e`, `perl -e`, `ruby -e`, `curl -o`, `wget -O`, `rsync`) con target Class A bloqueados (INFRA-H1); shell-subst (`$`, backtick, `~`) sin target literal Class A falla closed si el comando es write-shaped (SEC-M3). Read NO enforced (routing es "quién escribe", no "quién lee"). **enforce-secrets** (Read/Edit/Write/Bash — **aplica a todos los roles**, no hay bypass por marker): bloquea `.env*` (excepto `.env.example`, `.env.template`, `.env.local`, `.env.dev`, `.env.test`, `.env.sample`), `secrets/**`, `*.pem`, `*.key`, `*credential*` (case-insensitive). Bash verb detectors para read-style (`cat`, `less`, `head`, `tail`, `view`, `bat`, `tac`, `nl`, `od`, `hexdump`, `xxd`, `strings`) y edit-style (`sed -i`, `tee`, `cp`, `mv`, `perl -i`, redirecciones, `curl -o`, `wget -O`, `rsync`). **enforce-self-mod** (Edit/Write/Bash): protege **solo** la capa de enforcement — `.claude/hooks/**`, `.claude/settings.json`, `.claude/settings.local.json`, `.claude/.subagent-active.json*`, root-level `CLAUDE.md`. Los subdir `CLAUDE.md` (e.g. `.claude/agents/*/CLAUDE.md`, `playbooks/*/CLAUDE.md`) NO están protegidos — son per-agent docs. Bypass requiere subagente con `allow_self_mod=true` en el marker (e.g. `infra-devops-implementer`), no cualquier subagente. SEC-M3 shell-subst aplica igual. **enforce-subagent-isolation** (Edit/Write/Read/Bash, solo cuando role=subagent): confina al subagente al scope registrado en su marker. Shell-subst en cualquier valor falla closed. Fast-lane (`pretooluse-fast-lane.sh`) opera ortogonal como allow-refinement sobre Class A non-app para edits mecánicos ≤10L con exclusiones: self-mod (CLAUDE.md, .claude/**, playbooks/**, .cursor/**, .github/**), supply chain (package.json, lockfiles, docker-compose*, Dockerfile*, tsconfig*, Makefile, .npmrc, .yarnrc*), reviewer-trust (.gitignore, .gitattributes, .gitmodules), segmentos *auth*/*crypto*/*guard*/*secret*/*credential*. Case-insensitive matching (APFS). Symlinks via canonicalize. **Modes: `disabled` = full bypass; `hard` = default. `warn` NO se honra (SEC-M4) — honrar warn sería downgrade silencioso de hard-denies. Solo `disabled` permite bypass.** jq fail-closed: ausencia de jq aborta con exit 2 (SEC-L5). |

**Behavioral layer (primera línea)**:
- Skill invocation triggers (`superpowers:systematic-debugging`, `superpowers:verification-before-completion`, etc.) — soft gate, pero backed by structural layers.
- If behavioral layer fails, structural layers catch the issue.
